{% comment %} Include reutilizable para Grid.js con Vanilla JS. ================================================================================ USO BÁSICO ================================================================================ 1. Incluir el template con los parámetros requeridos: {% include "includes/_gridjs.html" with table_id="mi-tabla" url="/api/ingredientes/" columns="[ {name: 'ID', id: 'id', hidden: true}, {name: 'Nombre', id: 'name'}, {name: 'Unidad', id: 'unit'}, {name: 'Almacén', id: 'warehouse'}, {name: 'Stock', id: 'stock_quantity'} ]" %} 2. Opcionalmente configurar opciones adicionales: {% include "includes/_gridjs.html" with table_id="mi-tabla" url="/api/endpoint/" columns="[...]" searchable=true sortable=true pagination=true perPage=10 height="500px" column_filters=true filter_placeholder="Filtrar..." %} ================================================================================ PARÁMETROS DISPONIBLES ================================================================================ REQUIRED: --------- - table_id: (string) ID único para el elemento contenedor de la tabla Ejemplo: "ingredients-table" - url: (string) URL del endpoint API que retorna JSON La API debe retornar un array de objetos Ejemplo: "/api/ingredients/" - columns: (string JSON) Array con definición de columnas Cada columna es un objeto con: - name: (string) Nombre visible de la columna - id: (string) ID del campo en el JSON - hidden: (boolean) Ocultar columna (opcional) - width: (string) Ancho de columna (opcional) - formatter: (function) Función para formatear valores (opcional) Ejemplo: columns="[ {name: 'ID', id: 'id', hidden: true}, {name: 'Nombre', id: 'name'}, {name: 'Precio', id: 'price', formatter: function(cell) { return '$' + cell.toFixed(2); }} ]" OPCIONALES: ----------- - searchable: (boolean) Habilitar búsqueda global (default: true) - sortable: (boolean) Habilitar ordenamiento por columnas (default: true) - pagination: (boolean) Habilitar paginación (default: true) - perPage: (integer) Filas por página (default: 10) - height: (string) Altura del contenedor (default: 'auto') Ejemplo: "500px", "100%" - server: (boolean) Usar paginación/búsqueda del servidor (default: false) - column_filters: (boolean) Habilitar filtros por columna (default: true) Añade inputs de texto en cada columna para filtrar Usa lógica AND: filtra por TODAS las columnas con valores - filter_placeholder: (string) Placeholder de los inputs de filtro (default: "Filtrar...") ================================================================================ FORMATO DEL JSON DE LA API ================================================================================ La API debe retornar un array de objetos con los campos definidos en columns: [ {"id": 1, "name": "Arroz", "unit": "Libras", "warehouse": "Cocina", "stock_quantity": 100}, {"id": 2, "name": "Frijoles", "unit": "Libras", "warehouse": "Despacho", "stock_quantity": 50} ] ================================================================================ EJEMPLOS DE USO ================================================================================ Ejemplo 1 - Tabla básica: {% include "includes/_gridjs.html" with table_id="products" url="/api/products/" columns="[{name: 'Producto', id: 'name'}, {name: 'Precio', id: 'price'}]" %} Ejemplo 2 - Tabla con todas las opciones: {% include "includes/_gridjs.html" with table_id="ingredients" url="/api/ingredients/" columns="[ {name: 'ID', id: 'id', hidden: true}, {name: 'Nombre', id: 'name'}, {name: 'Stock', id: 'stock_quantity'} ]" searchable=true sortable=true pagination=true perPage=25 height="400px" column_filters=true filter_placeholder="Buscar en columna..." %} Ejemplo 3 - Con formateador de moneda: {% include "includes/_gridjs.html" with table_id="sales" url="/api/sales/" columns="[ {name: 'Producto', id: 'product_name'}, {name: 'Total', id: 'total', formatter: function(cell) { return '$' + cell.toFixed(2); }} ]" pagination=true perPage=50 column_filters=false %} ================================================================================ FILTROS POR COLUMNA (column_filters) ================================================================================ Cuando column_filters=true (default: true), se añade una fila de filtros: - Cada columna tiene un input de texto - Los filtros usan lógica AND: una fila se muestra si cumple TODOS los filtros - El filtrado es en tiempo real (se aplica al escribir) - Es case-insensitive (no distingue mayúsculas/minúsculas) - Busca coincidencia parcial en el texto Ejemplo: Si escribes "Arroz" en columna "nombre" y "Libras" en columna "unidad", solo se mostrarán las filas que cumplan AMBAS condiciones. {% endcomment %}
